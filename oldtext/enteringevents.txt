              <h1 class="ff-global-title">Entering events</h1>
              <div class="ff-post ff-post-editable ff-post-new">
                <div class="ff-post-content">
                  <div class="ff-container" id="ffContainer">
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_132480"><p>Scheduler can be used as a general purpose calendar, but be aware everything which you enter is potentially visible to all the other users. &nbsp;Try to make your event descriptions meaningful - rather than putting just "Meeting", put "Demonstration of Scheduler for teaching staff", or whatever it is that is happening. &nbsp;You may know what the meeting is about, but if it appears on someone else's schedule then they might well not.</p><p>Perhaps the first rule of entering an event is - check first to make sure it's not already there.  Since a typical event can involve lots of people and resources, it might be that someone has got there ahead of you.  Take a look at the proposed venue or people (see <a href="https://abingdon.fireflycloud.net/ict-and-elearning/scheduler/quick-tour">Quick tour</a>) to check that the event isn't already in the system. &nbsp;Much of the usefulness of the system will be lost if events are entered in a fragmented way. &nbsp;The rule is - each event is entered just once, and then the relevant resources are attached to the event.</p><p>You can start the entry of an event in one of several ways:</p><ul><li>Click on the calendar at the appropriate date and time.</li><li>Click and drag on the calendar, to set the duration as well.</li><li>Drag one of the coloured resource markers from the left hand margin on to the calendar. &nbsp;This will create an event, and add the requested resource to it.</li></ul><p>Whichever of these you do, you will be presented with a dialogue box something like this:</p></div>
                    <div data-ff-component-type="image" class="  ff-component-block even component ff-component ff-component-type-image" id="ffComponent_127359"><figure class="ff-image"><img src="./Entering events - Abingdon_files/resource(1).aspx" data-ff-nativeheight="997" data-ff-nativewidth="1258"></figure></div>
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_127367"><p>Note that the first box to be filled in asks for a description of what is happening - not who's going to be there or where it is; that information will come later. &nbsp;In the next shot I've filled in the description and chosen an event category of "Meeting". &nbsp;(Leave the Eventsource as Manual - this field is probably going to go away.) &nbsp;Since I just clicked on the calendar to start the process, my event has no duration. &nbsp;Both the Starts and Ends fields are set to the same time. &nbsp;I click on the Ends field to adjust the end time.</p></div>
                    <div data-ff-component-type="image" class="  ff-component-block even component ff-component ff-component-type-image" id="ffComponent_127368"><figure class="ff-image"><img src="./Entering events - Abingdon_files/resource(2).aspx" data-ff-nativeheight="997" data-ff-nativewidth="1258"></figure></div>
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_127402"><p>And here I can set the end time to 13:00. &nbsp;You can either use the graphical widgets to adjust the date and time, or type them directly into the text box at the top. &nbsp;I leave the Organiser field as me, because I am - if you're entering an event on behalf of someone else then put his or her name in this field. &nbsp;If it's on behalf of an external body (e.g. Henley Amateur Operatic Society) then this can go in the Reference field.</p><p>I can then click on Create Event, and the event record will be added to the database. &nbsp;It is now possible to associate resources with the event.</p></div>
                    <div data-ff-component-type="image" class="  ff-component-block even component ff-component ff-component-type-image" id="ffComponent_127380"><figure class="ff-image"><img src="./Entering events - Abingdon_files/resource(3).aspx" data-ff-nativeheight="997" data-ff-nativewidth="1258"></figure></div>
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_127386"><p>Note that a new field has appeared where you can enter the name of any type of resource - staff, pupil, location, group, service or property. &nbsp;The usual predictive text box appears and you choose from the list by pressing Enter. &nbsp;Once you have the value which you want in the field, press Enter again to add it to the event. &nbsp;Once I've added a couple of members of staff, the event looks like this:</p></div>
                    <div data-ff-component-type="image" class="  ff-component-block even component ff-component ff-component-type-image" id="ffComponent_127387"><figure class="ff-image"><img src="./Entering events - Abingdon_files/resource(4).aspx" data-ff-nativeheight="997" data-ff-nativewidth="1258"></figure></div>
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_137583"><p>Note how the system can tell different resources apart, and puts each in its correct column. &nbsp;Now I need somewhere to hold the meeting, so I'll ask for the Admin Hub Meeting Room. &nbsp;Again I type the name of the resource in the resource box, and press enter to add it. &nbsp;This time though, something different has happened. &nbsp;The item is highlighted in amber. &nbsp;This is because that room is managed and organised by Reception, and they need to check their bookings to ensure it is available.</p></div>
                    <div data-ff-component-type="image" class="  ff-component-block even component ff-component ff-component-type-image" id="ffComponent_127394"><figure class="ff-image"><img src="./Entering events - Abingdon_files/resource(5).aspx" data-ff-nativeheight="997" data-ff-nativewidth="1258"></figure></div>
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_127399"><p>The system uses a traffic light convention - Amber for checking, Green for OK and Red for not OK. &nbsp;Once the owner of the resource has approved your request the line will change to green, and if it has to be turned down then it will change to red, and you'll get an e-mail about it.</p></div>
                  </div>
