              <h1 class="ff-global-title">Schedule feed</h1>
              <div class="ff-post ff-post-editable ff-post-new">
                <div class="ff-post-content">
                  <div class="ff-container" id="ffContainer">
                    <div data-ff-component-type="html" class="  odd component ff-component ff-component-type-html" id="ffComponent_145654"><h3>Basic use</h3><p>To get a feed of your schedule (or indeed, the schedule of anyone or anything else) into an external calendar, you first need to work out the URL for the feed.</p><p>In it's simplest form, this will be just:</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/&lt;your initials&gt;</p><p>so for me, it would be:</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/jhw</p><p>You are strongly advised to use it as a <i>feed</i>,&nbsp;and not to try to copy the data over statically. &nbsp;That is - don't download a file and then upload it into your calendar program. &nbsp;Instead, put the URL directly into your calendar program and let it fetch over the information at regular intervals. &nbsp;This way the information will be kept up to date with changes as they occur.</p><p class="leading">Google Calendar</p><p>&nbsp;In Google Calendar you just click on the little down arrow to the right of "Other calendars", and choose "Add by URL". Then type in your chosen URL.</p><p class="leading">Apple iCal</p><p>On your laptop, start up Apple's Calendar program and use File =&gt; New Calendar Subscription to enter the URL of your feed. Link your laptop to your Apple ID (Calendar =&gt; Accounts) and this will then propagate to your iCloud and thus to your iPad - which should already be linked to your Apple ID.</p><h3>Advanced use</h3><p>If you use the simple URL shown in the previous section, you will get any details of any events in which you are involved (timetable, Other Half, cover, invigilation, calendar events), plus some overview events (week letter, term dates) all in a single feed. Unfortunately, Google Calendar provides no means then to separate them out and display them differently.</p><p>Personally I like to have my cover and invigilation events appear in red so they're immediately obvious. &nbsp;To achieve this, I need to use several separate URLs, each constructed to select particular categories of events.</p><p>To achieve this sort of thing, you add modifiers to the end of your URL.</p><p class="ff-style-guidance">The general form of these is dictated by the standards for HTTP (HyperText Transfer Protocol). You start by adding a question mark to the end of your URL, followed by your first modifier, and then if there are any more modifiers you introduce each one with an ampersand. &nbsp;Thus, if I wanted to add the modifiers, "able", "baker" and "charlie" to a URL it would read:&nbsp;http://abingdon.scheduler.org.uk/ical/jhw?able&amp;baker&amp;charlie</p><p>Modifiers which scheduler understands in the context of an ical feed are as follows:</p><ul><li>cover - provide only cover events in this feed</li><li>!cover - provide only events which are not cover in this feed</li><li>start_date= - specify a start date for the events</li><li>end_date= - specify an end date for the events</li><li>categories= - restrict the events in the feed to certain event categories</li></ul><p>If you specify any modifiers on the URL, then the system will stop giving you the overview events in that feed and you will have to request them separately. &nbsp;The URL for this is:</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/0</p><p>These can be combined to achieve quite a range of results. I use the following feeds for getting my events into Google Calendar.</p><p class="leading">For the overview events:</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/0</p><p class="leading">For my normal events:</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/jhw?!cover&amp;categories=!Invigilation</p><p>This specifies that I want events for me, except cover events and events which fall into the category "Invigilation".</p><p class="leading">For my cover and invigilation</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/jhw?cover</p><p>This specifies I want any cover events for me.</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/jhw?categories=Invigilation</p><p>And this one gives me any invigilation slots.&nbsp;</p><p>By taking all these as separate feeds, I can then choose to have them appear in different colours in Google Calendar.</p><p>Should you want to use the date modifiers above, the date should be given in ISO format - so the 16th Nov 2015 would be given as 2015-11-16.</p><p class="ff-style-code">http://scheduler.abingdon.org.uk/ical/jhw?start_date=2015-11-16</p></div>
